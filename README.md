#### 📅 2022.10.05

##### 1) 생성자
  - 객체 생성시 변수나 상수 초기화, 다른 초기화 기능을 수행하는 메소드를 호출함
  - 클래스에는 반드시 하나 이상의 생성자 존재

##### 2) this
  - 인스턴스 자신의 메모리를 가리킴.
  - 생성자에서 또 다른 생성자를 호출할 때 사용
  - 자기 자신 반환할 때 사용

##### 3) 접근 제어 지시자
  - public: 누구나 접근가능
  - default: 같은 패키지 내에서 접근가능
  - protected: 상속관계에서만 접근가능
  - private: 내 클래스 내부에서만 접근가능

##### 4) getter, setter
  - 접근 제어 지시자가 private일 경우 해당 변수에 접근하기 위해 사용
  - getter는 값을 확인할 때, setter는 값을 수정할 때 사용

#### 📅 2022.10.14

##### 1) 추상클래스
  - 구현 코드 없이 메서드의 선언만 있는 추상메서드를 포함한 클래스

##### 2) 템플릿 메서드 패턴
  - 템플릿 메서드는 추상메서드나 구현된 메서드를 활용하여 코드의 흐름을 정의하는 메서드

##### 3) final
  - 변수: 값이 변경될 수 없는 상수
  - 메서드: 하위 클래스에서 재정의 할 수 없는 메서드
  - 클래스: 상속할 수 없는 클래스

##### 4) 인터페이스
  - 모든 메서드가 추상메서드로 선언됨.
  - 모든 변수는 상수로 선언됨.
  - 구현된 것이 아무것도 없는 밑그림만 있는 설계도면
  - 상속받기위해 "implements" 사용

#### 📅 2022.10.17 ~ 2022.10.19

##### 1) Object 클래스
  - 모든 클래스의 최상위 클래스
  - 모든 클래스는 Object를 자동으로 상속받고, Object 클래스의 메서드 중 일부 재정의 가능

##### 2) 프로그램 오류
  - 컴파일 오류: 프로그램 코드 작성 중 발생하는 문법적 오류
  - 실행 오류: 실행 중인 프로그램이 의도하지 않은 동작을 하거나 프로그램이 중지되는 오류

##### 3) 에러와 예외
  - 에러는 가상머신에서 발생하는 것으로 프로그래머가 처리 X
  - 예외는 프로그램에서 발생하는 것으로 프로그래머가 처리 O

##### 4) 예외처리
  - 프로그램의 비정상 종료를 피하여 원할한 실행 유지
  - log를 남겨 추후에 log 분석을 통해 원인 파악 후 bug 수정 가능
  - Exception 클래스 사용
  - try-catch문 사용
    - try: 예외가 발생할 수 있는 코드 작성
    - catch: try부분에 예외 발생 시 처리하는 부분 작성
    - finally: 예외가 발생여부와 관계없이 try문이 실행되면 반드시 실행
  - throw를 사용해서 예외 처리 미루기 가능

#### 📅 2022.10.28 ~ 2022.10.31

##### 1) 익명 구현 클래스
  - 익명 클래스
    - 이름없는 클래스
    - 변수명이 없기 때문에 다른 메서드, 클래스 사용 X
    - 딱 그 순간에 필요시 사

##### 2) 어뎁터 패턴
  - 클래스의 인터페이스를 사용자가 기대하는 다른 인터페이스로 변환하는 패턴

##### 3) Thread
  - 하나의 프로세스는 하나의 이상의 Thread를 가짐
  - 실제 작업을 수행하는 단위
  - Thread = 작업
  - 자신만의 자원ㅇ르 가질 수 있고, 메인 쓰레그가 가지고 있는 자원도 공유 받아 사용가능
  - 자원 공유 시 동기화 처리 필수
  - Thread 종류
    - singleThread: Thread가 하나임.
    - multiThreading: 여러개의 Thread가 동시에 수행되는 프로그래밍, 마치 작업이 동시에 진행되는 효과를 얻음.

#### 📅 2022.11.07

##### 1) java 입출력
  - 대상 기준
    - 입력 스트림: 대상으로부터 자료를 읽어 들이는 스트림
    - 출력 스트림: 대상으로 자료를 출력하는 스트림
  - 자료의 종류
    - 바이트 단위: 동영상, 음악파일, 실행파일 등의 자료를 읽고 쓸 때
    - 문자 단위: 인코딩에 맞게 2바이트 이상으로 처리하도록 구현됨.
  - 기능
    - 기반 스트림: 대상에 직접 자료를 읽고 쓰는 기능
    - 보조 스트림: 직접 읽고 쓰는 기능은 없는 추가적인 기능을 더해줌.
